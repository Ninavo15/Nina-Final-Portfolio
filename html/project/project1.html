YumMeal Website

An all-in-one meal planning web app for finding, sharing, and saving recipes.

Overview

YumMeal is a full-stack recipe platform that allows users to search recipes (provided by the Spoonacular API), browse community recipe submissions, favourite recipes, write reviews, create and share their own recipes, and plan weekly meal schedules. The application also supports administrative features such as user management, team recipe creation, and content moderation, with role-based access enforced through authentication and protected routes.

Why This App

My team built YumMeal out of a frustration we kept running into ourselves: staring at random ingredients in the fridge, searching recipe after recipe, and never landing on something that actually matched what we had — then forgetting the recipes we did like by the next time we needed them. YumMeal exists to fix that: search by the ingredients already on hand, save the recipes worth returning to, and stop starting from zero every time you cook.

Outcome

Building YumMeal expanded my understanding of web development as a whole. One of the biggest lessons was learning how to store and handle data correctly — structuring it so it stays organized, and making sure it can be retrieved and modified safely. It also gave me a much clearer picture of how the frontend and backend actually work together, from authentication to API calls, rather than treating each side as its own isolated piece.

Tech Stack

Frontend

  • React 19 (react, react-dom) — bootstrapped with react-scripts (Create React App)
  • React Router v7 (react-router-dom) — client-side routing, plus custom ProtectedRoute / AdminProtectedRoute guards
  • jwt-decode — reads JWT payload client-side for auth context
  • lucide-react + remixicon — icon libraries
  • Testing Library (@testing-library/react, jest-dom, user-event) — component tests
  • Plain CSS per component/page (no CSS-in-JS or Tailwind/Bootstrap) — one .css file per .js component

Backend

  • Node.js + Express 5
  • MongoDB + Mongoose 9 — models: User, Favourite, MealPlan, Review, TeamRecipe, UserRecipe2
  • jsonwebtoken — auth token issuing/verification (authMiddleWare.js, verifyAdmin.js)
  • bcryptjs — password hashing
  • cors, dotenv
  • Spoonacular API — recipe search & ingredient data

Code

YumMeal’s finalized schema structure

  • User — contains user login information
  • UserRecipe — user-created recipes
  • TeamRecipe — team-created recipes
  • Favourite — users’ favourite recipe information
  • Review — reviews created by users
  • MealPlan — users’ meal plan information
