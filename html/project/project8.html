Scripting

  • JavaScript one script per feature in js/: cart.js, checkout.js, products.js, 3pro.js, script.js
  • No build tooling — no bundler/compiler config, no package manager
  • No backend — a purely client-side static storefront

Key Features

Dynamic Product Routing products.js, script.js

All six products live in one shared array (products.js). Clicking a product card — from the full shop grid or the home page’s first-three featured picks (3pro.js) — stores that product in localStorage and routes to a single product_info.html template, which reads it back and renders the detail view. Every product page is the same file, populated dynamically instead of duplicated per product.

Cart Built on localStorage cart.js

The cart is an array of product objects in localStorage — one entry per unit rather than a quantity field. cart.js groups entries by product id to render quantity and per-line totals; the +/− icons push or splice entries and recompute the grand total live, and clearing the cart shows an empty-state view linking back to Home or Shop.

Accessible by Default script.js, 3pro.js, cart.js

Every interactive element — product cards, quantity steppers, remove icons — carries role="button", tabindex="0", and a descriptive aria-label, with matching keydown handlers so Enter/Space mirror clicks. The cart icon’s label updates with the live item count, and add-to-cart actions are announced to screen readers.

Minimal Checkout Flow checkout.js

With no backend to hit, checkout is deliberately simple: submitting the form clears the cart from localStorage and redirects to confirmation.html — a front-end simulation of a purchase rather than real payment processing.
